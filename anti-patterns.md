# 错误示范画廊：六种调整集，看每种方法被偏到哪里

全书第 2 章的 DAG 给出过"该控制谁、不该控制谁"的判据。第 3–9 章以"控制对的 29 协变量"为前提把九种方法各跑一遍。但教学经验告诉我们，**读到"不该控制中介"和"不该控制对撞节点"的读者大多记不住**——这两条警告在抽象层面太薄，需要看到错的具体长什么样、偏到哪里、偏多少，才能真正吸收。

这一篇就是把错的样子做出来给读者看。在 RHC 同一份数据上，故意拿不同的"错调整集"喂给 G 计算、IPW、AIPW 三种主流方法，对比正确版本与各种错误版本的 ATE 估计值。所有数字均来自 [code/run_extras.R](code/run_extras.R) 的真实运行输出。

## 总览表

|:--|:--:|:--:|:--:|:--:|:--:|
| **正确：29 协变量** | **0.0508** | **0.0438** | **0.0433** | **0.0140** | **16.9** |
| A. 几乎不调整（仅 sex） | 0.0752 | 0.0753 | 0.0753 | 0.0136 | 2.81 |

正确基线是 RD ≈ 0.04–0.05；六种错误下，估计值从轻微偏移（F）到方向翻盘（E）都有。下面逐条展开每种错的来源、机制、危险信号。

## 错 A：几乎不调整（只放性别）

```r
covs_wrong <- c("sex_bin")  # 只控制性别
m_y <- glm(death180_bin ~ rhc_bin + sex_bin, data = d, family = binomial)
m_a <- glm(rhc_bin ~ sex_bin, data = d, family = binomial)
# G/IPW/AIPW 全部得到 RD ≈ 0.075
```

**结果：三种方法都给出 RD = 0.075，与第 1 章粗差异完全一致。**

> [!WARNING]
> **诊断与机制**
>
> 这条错误最显然——研究者意识到"应该多控制点变量"但只放了一个。性别 sex 与 RHC 决策几乎独立（医生不按性别决定是否插管），与 180 天死亡也只有微弱关联。控制一个与混杂结构无关的变量等于没控制，估计值不动，全部退化为粗差异。
>
> **AIPW 的双重稳健性在这里救不了场**：双重稳健性的承诺是"两个模型有一个对就行"，但前提是两个模型都至少包含足够多的真实混杂源。当处理模型与结局模型都只用 sex，两个模型都缺关键变量，双重稳健性失效。
>
> **这条错误的诊断信号**：粗差异与调整后估计值几乎相同。出现这种情况要立刻怀疑调整集是否覆盖了核心混杂。

---

## 错 B：漏掉 APACHE 与 GCS（最强的两个混杂）

```r
covs_wrong <- setdiff(covs_correct, c("apache_score", "glasgow_coma_score"))
# G 给出 0.0568；IPW 0.0555；AIPW 0.0558
# 偏差量：约 +0.013（高估约 30%）
```

**结果：三种方法估计值从正确的 0.04 漂到 0.056，偏高约 30%。**

> [!WARNING]
> **诊断与机制**
>
> APACHE 与 GCS 是 ICU 严重度的核心信号——病情越重越容易被插管、病情越重越容易死亡。两条正向关联相乘等于正向混杂，遗漏后估计偏高。第 3 章模型 3 加进 APACHE 后 OR 从 1.38 跌到 1.18，已经在数字上演示过。
>
> **关键观察**：最大权重从 16.9 飙到 46.5。当倾向得分模型缺了关键变量，模型对极端 PS 值的诊断失真，IPW 把权重错放在不该放大的人身上，方差跟着膨胀。
>
> **这条错误的诊断信号**：移除某个变量前后，估计值变动 ≥ 1 个百分点 + 最大权重显著上升 → 该变量是核心混杂，不能省。

---

## 错 C：仅控制人口学（age + sex + edu）

```r
covs_wrong <- c("age", "sex_bin", "edu")
# G 给出 0.0788；IPW 0.0794；AIPW 0.0797
```

**结果：三种方法都估到 0.079，比正确值高出 0.036（约 80% 偏差）。**

> [!WARNING]
> **诊断与机制**
>
> 这是流行病学论文里非常常见的错误——研究者按"标准化"的思维放年龄、性别、教育这些"基础人口学变量"，忽略真正驱动处理决策的临床信号。在 ICU 场景下，决定是否插管的核心是 APACHE / GCS / 血压 / 肌酐这些急性病情指标，年龄性别教育的解释力远低于这些。
>
> 这条错误的偏差比 B 更大（0.036 > 0.013），因为 C 不仅遗漏 APACHE 与 GCS，还遗漏所有合并症与生理指标——剩下三个变量解释的混杂太少，几乎等同于 A。
>
> **诊断信号**：调整集只有人口学变量、没有疾病严重度或合并症 → 调整不足，估计接近粗差异。

## 错 D：控制处理后变量 z<sub>post</sub>

```r
# 合成一个明显处理后的变量：z_post = 0.6*A + 0.4*Y_proxy + 0.2*DNR + noise
d$z_post <- 0.6 * d$rhc_bin + 0.4 * y_proxy + 0.2 * d$dnr_bin + rnorm(n, 0, 0.3)
covs_wrong <- c(covs_correct, "z_post")
# G 给出 0.0140 (崩塌)；IPW 0.0874 (反向膨胀)；AIPW 0.0586；最大权重 200！
```

**结果：三种方法分歧极大。**G 计算崩塌到 0.014（被偏到几乎为零），IPW 反向膨胀到 0.087，AIPW 介于两者之间但 SE 翻了 2.5 倍，最大权重从 17 飙到 200。

> [!WARNING]
> **诊断与机制**
>
> 处理后变量 $z_{\text{post}}$ 同时被处理 $A$ 影响（系数 0.6）和被结局相关因素影响（系数 0.4）。把它放进结局模型里，模型会把"RHC → z<sub>post</sub> → 死亡"这条因果路径上的部分变异归因到 z<sub>post</sub> 头上，让 RHC 的系数被人为压低。这是**过度调整偏差** (overadjustment bias)。
>
> G 计算崩塌到 0.014：结局模型把因果链拦腰截断，反事实预测里 RHC 与 $\hat m$ 的关联被剥离，CATE 被低估近 70%。
>
> IPW 反向膨胀到 0.087：处理模型现在用 z<sub>post</sub> 预测 RHC，z<sub>post</sub> 含 RHC 自己的信息，倾向得分接近 0 或 1，权重爆炸到 200。
>
> AIPW 给出 0.0586：双路抵消起了部分作用，SE 0.0345 是正确版本（0.0140）的 2.5 倍，CI 远比正确版本宽。
>
> **诊断信号**：加入新协变量后最大权重数量级翻倍 + G 与 IPW 估计向相反方向漂移 → 高度可疑加进了处理后变量。

---

## 错 E：控制对撞节点 z<sub>collider</sub>（最戏剧化的失败）

```r
# 合成一个对撞节点：z_collider = 0.5*A + 0.5*Y + noise
d$z_collider <- 0.5 * d$rhc_bin + 0.5 * d$death180_bin + rnorm(n, 0, 0.4)
covs_wrong <- c(covs_correct, "z_collider")
# G 给出 -0.183；IPW -0.181；AIPW -0.188
```

**结果：三种方法都把 RD 从 +0.04 翻到 −0.19。RHC 从"有害"被错算成"显著保护"。**

> [!IMPORTANT]
> **诊断与机制**
>
> 对撞节点 $z_{\text{collider}}$ 同时受 RHC 与死亡影响，在 DAG 上画作 $A \to z \leftarrow Y$。**未控制时它是关闭的**——RHC 与死亡之间不会通过它产生关联；**一旦控制就被打开**——RHC 与死亡之间凭空多出一条非因果路径，路径方向与真实因果效应相反。
>
> 在 RHC 上具体演示：z<sub>collider</sub> 等于 RHC 与死亡的混合信号 + 噪声。在"z<sub>collider</sub> 取值相同"的子群里，那些被插了管的人比那些没被插管的人更可能在死亡上"反向调整"——因为 z 的值若主要由 RHC 决定，则 Y 必须更小才能维持 z 平衡，反之亦然。这种"在 z 内部比较"凭空制造出 RHC 保护性效应。
>
> **这是因果推断中最戏剧化的错误**：从 +0.04 翻到 −0.19，方向反转、95% CI 完全不重叠，临床政策结论被颠倒。一份认真做的论文若不慎控制了对撞节点，会得出"RHC 是保护因素，应当推广"的相反建议。
>
> **诊断信号**：加入某个变量后估计方向反转 → **几乎一定是对撞节点**。这条信号比任何其他诊断都要可靠，遇到立即停下来回 DAG 检查节点角色。

## 错 F：APACHE 二值化（函数形式误设）

```r
# 把连续 APACHE 替换为二值化的 apache_high (>= 50 vs < 50)
d$apache_high <- as.integer(d$apache_score >= 50)
covs_wrong <- c(setdiff(covs_correct, "apache_score"), "apache_high")
# G 给出 0.0585；IPW 0.0522；AIPW 0.0505
```

**结果：估计值从正确的 0.04 漂到 0.05–0.06，偏高约 0.01（25%）。**

> [!WARNING]
> **诊断与机制**
>
> 与前几个错误相比，函数形式误设的偏差幅度最小、最不显眼。APACHE 是连续的严重度指标，把它二值化等于宣称"49 分与 30 分病情一样、50 分与 80 分病情也一样"。模型对中段与高段的混杂吸收能力下降，部分混杂残留。
>
> AIPW 的偏差（0.0072）小于 G 计算（0.0077）和 IPW（0.0089），双重稳健性给出了一定的保护——但**没有把偏差消到零**。原因是结局模型与处理模型用的是同一个误设函数形式，两个模型同错，残差乘积不再为零。
>
> **诊断信号**：连续协变量被二值化、对数变换被忽略、交互项缺失 → 隐性偏差在 0.01 量级。这种偏差比 D 与 E 难发现，唯一办法是用机器学习（DML / TMLE，第 7 章）做敏感性比较——如果灵活模型的估计与参数模型估计差超过 SE 的 2 倍，怀疑参数模型的函数形式有问题。

## 偏差的"量级地图"

把六种错误按偏差幅度从小到大排：

<div align="center">

| 错误类型 | 典型偏差 | 检测难度 | 处理建议 |
|:--|:--:|:--:|:--|
| F. 函数形式误设 | < 25% | 高 | 用 ML 做横向稳健性检查 |
| B. 漏掉强混杂 | 25–35% | 中 | DAG 论证调整集 + 敏感性分析 |
| C. 调整集不足 | 50–80% | 中 | DAG 论证 + 临床领域知识 |
| A. 几乎不调整 | ≈ 100% | 易 | 调整后估计与粗差异比较 |
| D. 控制处理后变量 | 方法间分歧大 | 中 | 检查变量是否在处理之后测量 |
| E. 控制对撞节点 | **方向反转** | 易（如果留意） | DAG 节点角色判别 |

</div>

> [!IMPORTANT]
> **DAG 是唯一可靠的判据**
>
> 六种错误中，A、B、C 三种都是"调整不足"，DAG 论证 + 领域知识可以避免；F 是"函数形式选错"，机器学习对照可以发现；最危险的是 D 与 E——**控制了不该控制的变量**——这两种错误在数据探索阶段几乎看不出来，唯一可靠的判别工具是 DAG。
>
> 第 2 章把"该控制谁、不该控制谁"讲清楚的真正用途在 D 与 E 这两种场景上——不画 DAG 就没法判断哪些变量是中介、哪些是对撞节点。看到一个变量"看起来与处理或结局相关"就放进调整集，是观察性研究里最容易踩的雷。

## 一份"调整集自检清单"

写论文之前，按这五条逐一过一遍：

1. **粗差异 vs 调整后估计：差距大不大？**差距小，调整集可能不足（错 A、错 C）。
2. **去掉单个变量看估计变化：哪些变量不可缺？**去掉后估计明显变动的变量是核心混杂（避免错 B）。
3. **每个协变量都是处理之前测量的吗？**任何在处理决策之后才生成的指标都不能进调整集（避免错 D）。
4. **每个协变量都不在处理与结局的下游吗？**下游变量是对撞节点，控制后会颠倒方向（避免错 E）。
5. **关键连续协变量有没有用线性以外的形式？**用 ML 方法对照参数模型，差距大说明函数形式可能错（避免错 F）。

把这份清单拿到任何因果推断论文上扫一遍，能在投稿前抓住 80% 的调整集错误。
