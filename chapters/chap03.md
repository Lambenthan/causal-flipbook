# 第 3 章 · 回归调整：从粗关联到条件关联

**本章目标**

- 用逐步加变量的逻辑回归观察 RHC 系数随调整集变化的轨迹
- 理解回归系数在因果推断语境下的含义、必要假设与局限
- 在 RHC 数据上得到全书第一个调整后的 ATE 估计

---

上一章的 DAG 已经替我们指出了哪些变量必须控制：年龄、性别、APACHE 评分、Glasgow 昏迷评分，以及若干合并症与生理指标。DAG 解决了"该控制谁"的问题，但还没回答"怎么控制"。这一章把控制这件事变成一行 R 代码。回归是研究者最熟悉的控制手段——把要调整的变量放到方程右边，让回归方程在它们的取值层上分别比较处理组与对照组。

我们要做的事情很朴素：从一个什么协变量都不放的"粗模型"开始，逐步往里加变量，盯着 RHC 的系数怎么动。如果某个变量加进去 RHC 系数明显往下走，意味着这个变量背的混杂被回归吸收掉了。漂移的方向、幅度、模式，本身就是混杂结构的现场快照。看清楚回归能走到哪一步、在哪一步会踩空，才能理解后续章节为什么还要引入 G 计算、倾向得分、双重稳健估计——后面这些方法都是对回归某个具体弱点的改进。

## 3.1 从粗关联到条件关联

第 1 章已经给出过那个引发 Connors 论文争议的数字：RHC 组的 180 天死亡率比非 RHC 组高 7.5 个百分点。把这个比较放进逻辑回归框架，得到的就是只含一个处理变量的最简模型。在掰开模型之前，先把模型用到的两个统计概念铺一遍：**几率**和**比值比**。

"几率"不是"概率"——几率算的是"死亡的人 ÷ 存活的人"。RHC 数据里非 RHC 组 3551 人，1650 人在 180 天内死亡、1901 人存活，死亡几率是 1650÷1901 ≈ 0.87；RHC 组 2184 人，1179 人死亡、1005 人存活，死亡几率是 1179÷1005 ≈ 1.17。**比值比 OR** 就是两组几率相除：1.17 ÷ 0.87 ≈ 1.35。意思是"RHC 组的死亡几率是非 RHC 组的 1.35 倍"——这只是粗关联，还不是因果效应。

OR 与百分比看起来像，但它衡量的是"几率"的相对变化，不是"概率"的相对变化。在罕见结局下两者数值近似；在常见结局下差异变大。RHC 数据里 180 天死亡率 50% 上下，不算罕见，OR 与风险差读起来不同——这条提醒我们后面非压缩性那一节会再回来。

逻辑回归的最简模型把这件事写成一行公式。

> **定义 3.1 — 粗比值比**
>
> 在逻辑回归 $\mathrm{logit}\,P(Y=1) = \beta_0 + \beta_1 A$ 中， $\exp(\beta_1)$ 是处理组相对于对照组的粗比值比，简称 crude OR。它度量的是未经任何协变量调整的处理-结局关联强度。
>
> 其中 $A$ 是处理变量，RHC 里 $A = 1$ 表示插了导管、 $A = 0$ 表示没插； $Y$ 是结局变量， $Y = 1$ 表示 180 天内死亡、 $Y = 0$ 表示存活； $\mathrm{logit}\,P = \log\frac{P}{1-P}$ 把概率压到对数几率尺度上，这样 $\beta_1$ 就有"几率比的对数"的解释， $\exp(\beta_1)$ 反推回 OR 数值。

粗 OR 和真正的因果效应之间隔着整个混杂结构。如果 RHC 的使用与病情严重程度相关，而病情严重程度又直接影响死亡率，那么粗 OR 里既有 RHC 本身对死亡的影响，也有"重症更容易插管、重症也更容易死"这条混杂路径的贡献。回归调整的逻辑是：把混杂变量也放到方程右边，让回归方程在每一个协变量取值层上分别比较处理组和对照组的结局，把混杂路径"堵住"。

但堵得要对。回归对协变量的处理方式是有具体形式的，通常是线性项，比如把 APACHE 评分写成 $\beta_2 \cdot \mathrm{apache}$。这等于让模型相信"评分每增加 1 分，对数死亡几率增加固定的 $\beta_2$"。如果真实关系是非线性的，比如评分从 10 涨到 20 几乎不影响死亡率、但从 50 涨到 60 死亡率陡升，单个线性项无法刻画这种弯曲的关系，混杂还是会从没调整干净的部分里残留下来。这种偏差叫**模型设定误差**，是回归在因果场景下最常见的失误，本章 §3.5 会展开。

## 3.2 逐步加变量：观察系数漂移

下面用 RHC 数据拟合四个嵌套模型。**模型 1** 是粗模型，只放 RHC 一个变量；**模型 2** 加入人口学变量年龄和性别；**模型 3** 在模型 2 基础上加入疾病严重度的两个核心指标——APACHE 评分和 Glasgow 昏迷评分；**模型 4** 进一步加入合并症与全部生理指标，共 33 个协变量。

每次加变量我们只盯一个数字：RHC 系数对应的 OR。如果某一步加进去 OR 明显变了，就说明新加的变量在背一部分混杂——它原来藏在 RHC 系数里、现在被自己的位置上接住了。如果 OR 几乎不动，则这个变量与 RHC 或与死亡的关联很弱，背不动什么混杂。

```r
set.seed(2026)
library(tidyverse)
library(broom)

d <- read_csv(here::here("data", "rhc.csv"), show_col_types = FALSE) |>
  mutate(death180_bin = if_else(death180 == "Yes", 1L, 0L),
         sex_bin      = if_else(sex == "Male", 1L, 0L))

# 模型 1：粗模型，只看 RHC 与死亡的边际关联
m1 <- glm(death180_bin ~ rhc, data = d, family = binomial)

# 模型 2：加入人口学——年龄和性别本身是混杂还是精度变量？
m2 <- glm(death180_bin ~ rhc + age + sex_bin,
          data = d, family = binomial)

# 模型 3：加入疾病严重度——APACHE 和 GCS 是 ICU 决策的核心信号
m3 <- glm(death180_bin ~ rhc + age + sex_bin +
            apache_score + glasgow_coma_score,
          data = d, family = binomial)

# 模型 4：加入合并症 + 全部生理指标
m4 <- glm(death180_bin ~ rhc + age + sex_bin +
            apache_score + glasgow_coma_score +
            cancer + cardiovascular + congestive_hf + dementia +
            pulmonary + renal + hepatic + blood_pressure +
            heart_rate + respiratory_rate + temperature +
            albumin + creatinine + bilirubin + wbc + hematocrit +
            das_index + dnr_status + medical_insurance + race +
            income + edu + transfer_hx + mi + gi_bleed +
            tumor + immunosupperssion + psychiatric,
          data = d, family = binomial)

# 提取四个模型中 RHC 的 OR 和 95% CI
bind_rows(
  tidy(m1, conf.int = TRUE, exponentiate = TRUE) |> filter(term == "rhc") |> mutate(model = "Model 1"),
  tidy(m2, conf.int = TRUE, exponentiate = TRUE) |> filter(term == "rhc") |> mutate(model = "Model 2"),
  tidy(m3, conf.int = TRUE, exponentiate = TRUE) |> filter(term == "rhc") |> mutate(model = "Model 3"),
  tidy(m4, conf.int = TRUE, exponentiate = TRUE) |> filter(term == "rhc") |> mutate(model = "Model 4")
) |> select(model, estimate, conf.low, conf.high)
```

> **结果解读**
>
> 四个模型给出的 RHC 几率比和 95% 置信区间如下表。

<div align="center">

| 模型 | OR | 95% CI | 新增协变量 |
|:--|:--:|:--:|:--|
| Model 1 | 1.35 | [1.21, 1.50] | 无 |
| Model 2 | 1.38 | [1.24, 1.54] | age, sex |
| Model 3 | 1.18 | [1.05, 1.33] | apache_score, glasgow_coma_score |
| Model 4 | 1.34 | [1.18, 1.52] | 合并症 + 全部生理指标 |

</div>

这组数字里藏着几条信息。从模型 1 到模型 2，OR 从 1.35 微升到 1.38。年龄和性别加进去几乎没动 RHC 系数，说明这两个变量在 ICU 决策中的混杂作用不大——它们当然预测死亡率，但预测"谁被插管"的能力很弱。

真正的变化发生在模型 3。加入 APACHE 评分和 Glasgow 昏迷评分之后，OR 从 1.38 一步降到 1.18，降幅超过 14%。这与临床预期一致：APACHE 是 ICU 里决定是否插管的核心信号，它同时强烈预测死亡率，是典型的正向混杂变量。控制了它，RHC 与死亡之间一大块虚假关联被剥离了。

**停下来想一想。** 加入 APACHE 后 OR 从 1.38 降到 1.18。这等于在告诉你：原本 RHC 与死亡率之间那 38% 的几率比里，有大约 20 个百分点是 APACHE 评分自己的"锅"——重症患者更容易被插管，重症患者也更容易死亡，这条混杂路径贡献了相当一部分粗关联。控制 APACHE 之后，剩下的 18% 才更接近 RHC 本身在做的事情。

模型 4 加入合并症之后，OR 又回升到 1.34，几乎弹回粗模型水平。这个反弹说明**多放变量不等于更干净**。合并症变量里有些可能位于 RHC 与死亡的因果路径上、起中介作用，有些可能与处理后状态相关、起碰撞因子作用，还有些可能只是引入了新的非线性遗漏。无论原因是哪一种，系数的非单调漂移都是同一个提醒：回归调整的结果高度依赖你放了什么变量、用了什么函数形式。

## 3.3 系数漂移背后的混杂吸收

为什么加入不同协变量会让 RHC 系数往不同方向移动？答案在"偏回归系数"的几何意义里。先把这个意义在 RHC 数据上掰开。

模型 2 的粗 OR 是 1.38，看上去 RHC 把死亡几率拉高了 38%。但 APACHE 评分高的患者既更容易被医生选去插管（血流不稳的病人需要监测），又本来就更容易死亡（病情严重）。如果不把 APACHE 也放进比较里，"RHC → 死亡"这条粗关联里就掺了"APACHE 高 → 被插管 + APACHE 高 → 容易死"的混杂成分。当我们在模型 3 把 APACHE 加进去、让回归在 APACHE 评分相同的两组患者之间再比较 RHC vs 非 RHC，OR 从 1.38 降到 1.18，降幅约 14%。这 14% 就是 APACHE 在自己的位置上接住了原本被 RHC 系数背着的混杂——这就是**混杂吸收**。

下面这段 GIF 把这件事画成一个三节点 DAG 加一行遗漏变量偏差公式：箭头从 APACHE 同时通向 RHC 和死亡，偏差量 $\gamma\delta$ 标在 RHC 的系数旁边，加了 APACHE 后偏差被"按"回到 APACHE 自己的格子里。

<div align="center">
  <img src="../assets/chap03/03_confounding_absorption.gif" alt="混杂吸收：APACHE 把偏差从 RHC 系数里接走" width="720" />
</div>

为什么偏差会"自动"流向新加的变量？**Frisch-Waugh-Lovell 定理**给出了一个清楚的答案。在线性回归里， $A$ 的偏回归系数等于先把 $A$ 和 $Y$ 各自对其他协变量 $L$ 做回归取残差，再用 $A$ 的残差预测 $Y$ 的残差。也就是说，偏回归系数衡量的是"剔除 $L$ 能解释的部分之后， $A$ 中剩余的变异和 $Y$ 中剩余的变异之间的关系"。当模型加入一个新变量时， $L$ 的范围扩大、被剔除的部分更多、 $A$ 与 $Y$ 的残差都跟着改变，系数自然跟着变。

> **定义 3.2 — 混杂吸收**
>
> 如果变量 $L$ 同时影响处理 $A$ 和结局 $Y$，且 $L$ 未被控制，那么 $A$ 的回归系数会"吸收"一部分 $L \to Y$ 的效应。把 $L$ 加入模型后， $A$ 系数的变化量就是被吸收的混杂被释放出来的量。变化的方向取决于 $L$-$A$ 关联与 $L$-$Y$ 关联的符号组合。

最简单的判断方向有一个口诀。设真实模型是 $Y = \beta_1 A + \gamma L + \varepsilon$，但你只拟合了 $Y = \tilde\beta_1 A + \varepsilon$。**遗漏变量偏差公式**说

$$\tilde\beta_1 = \beta_1 + \gamma \cdot \delta$$

其中 $\beta_1$ 是真实因果系数，意思是"在固定 $L$ 的层内 $A$ 对 $Y$ 的效应"； $\gamma$ 是 $L$ 对 $Y$ 的真实系数，意思是"病情每重 1 分，死亡几率涨多少"； $\delta$ 是 $L$ 对 $A$ 回归得到的系数，意思是"病情每重 1 分，被插管的概率涨多少"； $\tilde\beta_1$ 是遗漏 $L$ 时算出来的偏值系数。 $\gamma$ 和 $\delta$ 同号时偏差为正、粗系数偏高；异号时偏差为负、粗系数偏低。这个公式严格只在线性模型成立，但方向上的判断在逻辑回归里近似适用。

> **命题 3.1 — 遗漏变量偏差的方向判断**
>
> 在线性回归里，遗漏变量 $L$ 造成的偏差方向，由 $L$-$A$ 关联与 $L$-$Y$ 关联两条符号的乘积决定。正乘正或负乘负产生正向偏差，让处理效应估计偏高；正乘负或负乘正产生负向偏差。在逻辑回归这类非线性模型中规律近似成立，可用来预判加入某个协变量后系数会往哪个方向走。

把这个口诀带回 RHC。APACHE 评分与 RHC 使用正相关，因为病情越重越可能被插管；与死亡率也正相关，因为病情越重越可能死亡。两条正关联相乘等于正向混杂，原本粗 OR 偏高 14%，控制 APACHE 后这部分偏差被剥离，RHC 的 OR 下降到 1.18。

模型 2 中加年龄微升的现象也能用同样的公式解释。ICU 实际决策逻辑是复杂的：年轻而重症的患者，医生倾向积极干预；高龄而重症的患者，医生和家属可能选择更保守的路径。这意味着年龄和 RHC 使用之间可能存在**负**关联。同时年龄与死亡率正相关。负乘正等于负向偏差，所以粗模型里年龄造成的混杂方向是把 OR 拉低。控制年龄之后这部分负向混杂被移走，OR 从 1.35 微升到 1.38。从 1.35 到 1.38 幅度小，这与"年龄在 RHC 决策中权重较低"的临床事实一致。

模型 4 系数反弹更值得警惕。OR 从 1.18 跳回 1.34，几乎回到粗模型水平。一种可能是某些合并症变量位于 RHC 与死亡的因果路径上：RHC 的使用有可能改变患者后续被诊断出某些并发症的概率，控制这些"处理后变量"等于控制中介，会阻断本应看到的因果效应。另一种可能是 33 个变量里有不少高度相关，比如肌酐与肾病史、APACHE 与若干生理指标，多重共线性会让回归面的倾斜变得不稳定，系数估计跟着抖。第三种可能是某些合并症与 APACHE、肌酐这些已经在的变量构成"调整碰撞因子"的局部 DAG 模式，加进去反而打开了新的非因果路径。

回归没法自动判断哪些变量该放、哪些不该放，它只负责在指定的模型下计算条件关联。DAG 的价值就在这里：DAG 告诉研究者调整集，回归只是执行这一调整的计算工具。没有 DAG 给出的调整集，回归出来的系数就缺乏因果解释的依据，研究者也没有办法判断结果是否对应真正的因果效应。

## 3.4 条件 OR 与边际 OR：非压缩性

回归读出来的 OR 还有一个容易被忽略的细节。即使你的调整集完全正确、模型设定也完全无误，**回归给的 OR 仍然不是你最关心的那种 OR**。这一节要讲的现象与混杂无关，纯粹是 OR 这个度量本身的数学性质——**非压缩性**。

先用一个数字例子说明这件事。假设有两个 ICU 病房，每个病房各 100 名患者。

**病房 A**，基线死亡率高：RHC 组死亡概率 0.80，对照组 0.60。条件 OR = $\frac{0.80 / 0.20}{0.60 / 0.40} = \frac{4.00}{1.50} = 2.67$。

**病房 B**，基线死亡率低：RHC 组死亡概率 0.40，对照组 0.20。条件 OR = $\frac{0.40 / 0.60}{0.20 / 0.80} = \frac{0.67}{0.25} = 2.67$。

两个病房的条件 OR 完全相同，都是 2.67。把两个病房的 200 名患者混在一起算总体的边际 OR：RHC 组总死亡概率 0.60，对照组总死亡概率 0.40，边际 OR = $\frac{0.60 / 0.40}{0.40 / 0.60} = \frac{1.50}{0.67} = 2.25$。

同样的条件效应，**混合之后数值变小了**。这不是混杂造成的，因为病房 A、B 内部数字都是给定真实因果效应的；纯粹是 OR 在跨子群合并时的数学性质。

  <img src="../assets/chap03/04_non_collapsibility.gif" alt="非压缩性：条件 OR 2.67 → 边际 OR 2.25" width="720" />
</div>

> **定义 3.3 — 非压缩性**
>
> 设条件 OR 为 $\mathrm{OR}_L = \exp(\beta_1)$，边际 OR 为在全人群上的几率比。即使不存在任何混杂， $\mathrm{OR}_L$ 与边际 OR 通常不相等。这种现象叫非压缩性，英文称 non-collapsibility，是 OR 这个度量本身的数学性质，与混杂无关。

直观地理解为什么会这样：把两个病房内部的"死亡率差异都是 20 个百分点"合并起来仍然是 20 个百分点，风险差在加权平均下保持不变。但 OR 是几率的比，几率是非线性的；非线性变换在加权平均下不可交换，所以混合两个病房之后 OR 会改变——条件 OR 都是 2.67，混合后边际 OR 缩到 2.25。

非压缩性带来的实际后果是：模型 4 读到的 OR = 1.34 不能直接解读成"如果所有人都接受 RHC，死亡几率比不接受的高 34%"。这是一个**条件**几率比的"内部读数"，回答的是"在协变量层完全相同的两个人之间比较"；而你大概率想要的是**边际**几率比，"如果整个人群都接受 RHC vs 整个人群都不接受 RHC"。两者之间的差距与混杂无关，是 OR 这个度量天生的特性。

风险差和风险比则没有非压缩性问题。这也是为什么越来越多的因果推断方法学家建议把因果效应报成风险差或风险比，而非 OR。下一章的 G 计算就会用标准化的方式直接在概率尺度上算出全人群的边际风险差，绕过非压缩性。

## 3.5 回归的因果解读条件

漂移让我们看到了混杂被吸收的过程，非压缩性提示了 OR 解读的微妙之处，但还有一个更根本的问题没回答：**即便调整集完全正确，回归系数在什么条件下才等于因果效应？**

> **定义 3.4 — 回归因果解读的充分条件**
>
> 在逻辑回归 $\mathrm{logit}\,P(Y=1 \mid A, L) = \beta_0 + \beta_1 A + \boldsymbol{\beta}_L^\top L$ 中， $\exp(\beta_1)$ 等于条件因果几率比的充分条件包含三条：
>
> **可交换性** $Y(a) ⫫ A \mid L$，即调整集 $L$ 足以阻断所有后门路径；
>
> **正值性** $0 < P(A=1 \mid L) < 1$，即每一个协变量层都有处理组也有对照组；
>
> **模型设定正确**，即 $\mathrm{logit}\,P(Y=1 \mid A, L)$ 在 $A$ 与 $L$ 上的真实关系恰好是上式那个线性形式。
>
> 其中 $\boldsymbol{\beta}_L$ 是协变量 $L$ 的系数向量， $\boldsymbol{\beta}_L^\top L$ 是把所有协变量与系数线性加权求和。 $Y(a)$ 是个体在处理 $A=a$ 下的潜在结果。

前两条假设是所有因果推断方法共享的——可交换性与正值性是因果识别的基石，无论用回归、G 计算还是倾向得分都要面对它们。回归独有的额外负担是**第三条：模型设定正确**。

模型设定有两个常见雷点。第一是非线性遗漏。本章的四个模型都把所有协变量以线性方式放入 logit 链接函数，没加交互项、没加非线性变换。如果 APACHE 评分对死亡率的影响存在阈值效应，比如评分 30 以下死亡率几乎不变、超过 30 后陡升，一个线性项就无法捕捉这种弯曲。下面这段 GIF 把"真实阈值曲线"与"线性回归拟合的直线"画在一起，标出在哪段会高估、在哪段会低估。

第二个雷点是交互项遗漏。如果 RHC 在年轻患者中是有害的、在老年患者中是有益的，把全样本拟合一个统一的 $\beta_1$ 会把这两种相反的效应平均成一个看起来"轻微有害"的中间数。这种情形叫**效应异质性**，第 9 章的因果森林会专门处理。

模型设定误差的麻烦在于它**隐性**。模型照样能收敛、照样会给出一个 OR 和一个 $p$ 值，研究者没有任何自动提示告诉他"你的函数形式错了"。Hosmer-Lemeshow 检验和 AIC 可以给一些间接信号，但它们检测的是整体拟合优度，对单个系数的偏误并不敏感。

这就是回归调整的根本困境：**它要求研究者事先猜对函数形式**。协变量少时可以靠领域知识加交互项与样条；协变量多到几十个时，两两交互的组合空间爆炸，手工设定模型变得不现实。RHC 数据有 49 个变量，光两两交互就有上千项，加上各种非线性变换，可能的模型空间大到无法穷举。后续每一种方法都是从不同角度对回归这个困境的回应——G 计算用标准化代替读系数；倾向得分把建模负担从结局端转到处理端；AIPW 同时建两个模型，只要一个对就稳；机器学习直接用数据驱动的方式拟合灵活函数形式。

> **雷区 — Table 2 谬误**
>
> 临床研究里常见一种做法：在论文表里把多元回归的所有协变量系数都列出来、逐一解读为各自的因果效应。这种做法被称为 Table 2 Fallacy。
>
> 问题在于：每个协变量的因果系数对应的调整集不同。当你解读年龄的系数时，模型已经控制了 APACHE 评分；但 APACHE 可能是年龄影响死亡率的中介——年龄通过病情严重度作用于死亡，控制 APACHE 等于切断了这条路径，得到的就不是年龄的总因果效应。同一个回归方程里，RHC 的系数可能需要控制 APACHE，年龄的系数则不应该控制 APACHE，两者要的 DAG 不同，不能共用一组结果。
>
> 安全的做法是每个因果问题对着自己的 DAG 画调整集，分别建模。把一个回归方程的全部系数当因果效应表来读，是因果推断里最常见的失误之一。

下面这段 GIF 用一张回归输出表演示 Table 2 谬误：每一行对应一个变量，但只有真正按 DAG 调整过的那一行能解读为因果效应，其他行只是回归内部的调整产物。

## 3.6 回归调整的局限

回归是因果估计里最容易上手的方法，但有几条绕不过去的局限。除了刚才讨论的模型设定敏感性，还有几个值得在动手之前知道。

回归没有显式构造反事实。它给出的是一个系数，研究者必须相信模型函数形式正确，才能把这个系数翻译成因果效应。下一章的 G 计算会为每个个体分别预测"接受 RHC"和"不接受 RHC"两个潜在结果，然后取差值——这种显式构造让反事实推理透明，模型诊断也直观。回归用户看到的只是一个数 1.34；G 计算用户看到的是 5735 个个体各自的两条预测，每一条都可以肉眼或诊断检验来核对。

回归对正值性违反没有内置警报。如果某一类患者几乎全部接受了 RHC，回归仍会给出一个系数，但这个系数在这个子群里几乎完全靠外推。考虑一个 APACHE 极高的层，里面 98% 都插了管，2% 没插。回归在这层里实际上比较的是 98 个处理样本和 2 个对照样本，统计功效极低，估计严重依赖线性外推。倾向得分方法至少可以通过查看得分分布发现这种重叠区缺失，回归则不会显式提示研究者哪些子群存在外推问题。

回归还假设处理效应在所有协变量层上是同质的。模型 $\mathrm{logit}\,P = \beta_0 + \beta_1 A + \boldsymbol{\beta}_L^\top L$ 只给 $A$ 一个系数，意味着 RHC 对年轻人和老年人、轻症和重症的效应被强制设为相同。如果真实效应是异质的，这个统一的 $\beta_1$ 会把异质性平均掉，得到一个看起来温和但其实可能误导政策的"平均"估计。

> **雷区 — 过度调整**
>
> "变量放得越多越安全"是研究新人最常见的错觉。在因果推断里，控制中介变量、碰撞因子或处理后变量都会引入新的偏差。模型 4 中合并症变量加进去后 OR 从 1.18 反弹到 1.34，其中一个可能的原因正是某些合并症位于因果路径上——控制它们等于切断 RHC 通往死亡的中介通道。
>
> 判断一个变量该不该放进模型，唯一可靠的依据是 DAG，而不是统计显著性、AIC 下降，或者"放了总比没放好"的判断。Angrist 与 Pischke 把这类问题称为"坏的控制变量"，它们看起来在控制混杂，实际上在制造偏差。

这些局限加起来，意味着回归调整的结论必须谨慎解读。但它仍然有用。本章四个嵌套模型已经显示了它的最佳用途：在正式建模之前快速揭示混杂结构，告诉研究者哪些变量是关键混杂源——比如 APACHE 一加进去 OR 就移动 14%，这条信息对后续所有方法都有指导意义。许多顶级期刊的因果推断论文也会在附表里报一个回归调整的结果作为基准线，再用更精细的方法做主分析。两者比较本身就是敏感性检查的一部分：如果差距很大，模型设定或非压缩性正在起作用，需要进一步排查。

| 章 · 方法 | ATE 估计 | 95% CI | 核心假设 | 局限 |
|:--|:--:|:--:|:--|:--|
| 第 1 章 · 粗差异 | RD = 0.075 | — | 无 | 未调整任何混杂 |
| **第 3 章 · 回归调整** | **OR = 1.34** | **[1.18, 1.52]** | **可交换性 + 正值性 + 模型设定正确** | **函数形式敏感、无显式反事实、无正值性诊断、效应同质性假设** |

这里报告的是模型 4，即放了 33 个协变量的全调整模型。OR = 1.34 意味着在控制了这 33 个变量后，接受 RHC 的患者 180 天死亡几率仍然比未接受者高 34%，95% CI 不包含 1， $p < 0.001$。结论方向与 Connors 1996 原始论文一致。但我们现在知道这个数字背后依赖的假设链很长：函数形式正确、调整集完整、效应同质，其中任何一条违反都会让 1.34 偏离真实因果效应。

回归给了我们一个可比较的基准。后续每一章的方法都会给出自己的 ATE 估计，最终我们看这些数字是否收敛。如果不同方法答案接近，因果结论的信心增强；如果答案分歧大，就说明某些假设可能被违反，需要进一步诊断。下一章的 G 计算与回归用同一个结局模型，但用标准化代替读系数，直接在概率尺度上构造全人群的反事实——读者将看到从"读系数"到"造反事实"会让 ATE 的数字与解读发生什么变化。

## 本章知识地图

<div align="center">

| 核心概念 | 核心内容 | 常见误解 | 为什么错 |
|:--|:--|:--|:--|
| **粗 OR** | 未调整任何协变量的处理-结局几率比 | 粗 OR 就是因果效应 | 混杂未被控制，关联不等于因果 |
| **混杂吸收** | 加入协变量后处理系数变化的量，反映混杂被回归吸收的过程 | 系数总是单调下降 | 方向取决于 $L$-$A$ 与 $L$-$Y$ 的符号组合，负向混杂会让系数上升 |
| **遗漏变量偏差** | $\tilde\beta_1 = \beta_1 + \gamma\delta$，遗漏 $L$ 时粗系数偏离真值的量 | 公式只对线性模型成立 | 方向上的判断在逻辑回归里近似适用，可用来预判系数移动方向 |
| **模型设定正确** | 回归的函数形式必须与真实数据生成过程一致 | 放了正确的变量就够了 | 变量正确但函数形式错误，比如线性遗漏或交互遗漏，仍会产生设定偏误 |
| **非压缩性** | 即使不存在混杂，条件 OR 与边际 OR 在非线性模型中通常不相等 | 调整后 OR 下降一定是混杂被控制 | OR 在跨子群合并时的数学性质本身就改变数值 |
| **Table 2 谬误** | 把回归输出表里所有系数都解读为因果效应 | 同一回归方程的所有系数共用一个调整集 | 每个变量的因果问题对应不同 DAG，不能共用一组结果 |
| **过度调整** | 控制中介、碰撞因子或处理后变量引入的新偏差 | 变量放得越多越安全 | 中介控制切断因果路径，碰撞控制打开新的非因果路径 |

</div>
