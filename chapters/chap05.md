# 第 5 章 · 倾向得分：匹配、加权与平衡诊断

**本章目标**

- 理解倾向得分的定义与降维定理为何能把 38 维匹配压缩成一维
- 在 RHC 数据上用逻辑回归估计倾向得分，并检查两组分布的重叠
- 掌握倾向得分匹配、逆概率加权两种使用方式及其平衡诊断
- 更新累积对比表，新增 PSM 与 IPW 两行因果效应估计

---

第 3 章逐步加变量地建结局回归，把 RHC 系数从 1.35 一路拉到 1.18 又被合并症拽回 1.34；第 4 章用 G 计算给每位患者填两列反事实结局，得到边际风险差 RD = 0.060。两章的赌注都压在同一个对象上：结局模型 $E[Y \mid A, L]$。RHC 数据有 38 个协变量，结局模型要不要给 APACHE 评分加二次项、要不要为年龄和合并症之间加交互、肌酐与血肌酐清除率之间该不该写非线性变换，每一项决策都没有标准答案。结局模型一旦设错，前两章方法没有第二条退路。

这一章把建模的赌注换一个对象——不去追问"如果给这位患者插管，他的 180 天死亡概率是多少"，而是反过来问"根据这位患者的基线特征，他被插管的概率是多少"。Rosenbaum 与 Rubin 1983 年证明，只要这个"被插管的概率"建对了，就可以用它把处理组和对照组拉成可比的两群人，结局模型一行都不用写。建模的负担从结局端整体迁移到处理端。这种迁移并不只是换一个被预测变量这么简单：处理变量在 RHC 数据里只有 0 或 1 两个取值，预测它远比预测连续生理结局来得稳定，而且模型对不对，可以用"匹配或加权之后协变量平不平衡"这个直接可观察的指标来诊断。

## 5.1 倾向得分：从一位 ICU 患者的"被插管概率"说起

先把场景具体到 RHC 数据里的一位患者。设想一位 50 岁、APACHE 评分 60、肌酐 1.5、收缩压 70 mmHg 的 ICU 患者。临床医生看着他的化验单，心里大致在做这样一件事：评分 60 已经是 RHC 数据里的中高位，肌酐 1.5 提示肾功能异常，收缩压 70 表明血流动力学不稳，综合起来，这位患者更像那群被插了右心导管的 RHC 组，而不像基本面平稳的对照组。如果让医生估个数，他会说"这个人被插管的概率大概八九成"。倾向得分要做的事，就是把"这个综合判断"用一个数学函数固化下来：输入患者的 38 个基线变量，输出一个 0 到 1 之间的数字。

把这件事用具体数字演示一遍。在 RHC 数据上拟合好倾向得分模型之后，把上面这位患者的协变量代进去，得到的预测概率大约是 0.85。再换一位 65 岁、APACHE 评分 25、肌酐 0.9、收缩压 130 的患者，他的预测概率约为 0.18。前者的 0.85 与后者的 0.18 把医生心里"这两位差很多"的临床直觉量化成了一个数。两位患者之间隔着 38 维协变量空间里的一段距离，倾向得分把这段距离压缩成数轴上 0.85 和 0.18 之间的差。

> [!NOTE]
> **定义 5.1 — 倾向得分**
>
> 给定基线协变量向量 $L$，个体 $i$ 的倾向得分定义为在协变量条件下接受处理的概率：
>
> $$ e(L_i) = P(A_i = 1 \mid L_i). $$
>
> 其中 $A_i$ 是处理变量，RHC 里 $A_i = 1$ 表示这位患者被插了右心导管、$A_i = 0$ 表示没插；$L_i$ 是这位患者的基线协变量向量，本章用的是第 2 章 DAG 给出的 38 维调整集，包括年龄、APACHE 评分、Glasgow 昏迷评分、血压、肌酐等；$e(L_i)$ 是这位患者根据自身基线特征"被插管的条件概率"，取值在 0 到 1 之间，纯粹由 $L_i$ 决定，不涉及结局 $Y$。

**回到 RHC 数据**。倾向得分把"38 维协变量"重新表述成"被插管概率"，这一步是一种语言翻译，本身还没有解决任何因果问题。真正让翻译有用的是下面这条降维定理。

## 5.2 降维定理：38 维协变量怎么压成一维

回归调整和 G 计算共同的困境是协变量维度。RHC 数据里有 38 个协变量，每位患者就是 38 维空间中的一个点。要在这个空间里直接为处理组找"双胞胎"对照，要求年龄相同、APACHE 相同、肌酐相同、血压相同、所有合并症都一样，这种完美邻居几乎不存在。维度越高，能完全相邻的点越少；分层做也不行，38 个二值变量就能切出 $2^{38}$ 个格子，比全部 5735 个样本多出十亿倍，绝大多数格子里一个人都没有。

Rosenbaum 与 Rubin 在 1983 年那篇 *Biometrika* 论文里给出了一条数学结果：你不需要在 38 个变量上同时匹配，只需要在 $e(L)$ 这一个数字上匹配就够了。原话写成数学是这样：如果可交换性在完整协变量 $L$ 上成立，即 $Y(a) \perp\!\!\!\perp A \mid L$，那么可交换性在标量倾向得分 $e(L)$ 上同样成立，即 $Y(a) \perp\!\!\!\perp A \mid e(L)$。

> [!NOTE]
> **定义 5.2 — 降维定理**
>
> $$ Y(a) \perp\!\!\!\perp A \mid L \;\;\Longrightarrow\;\; Y(a) \perp\!\!\!\perp A \mid e(L). $$
>
> 其中 $Y(a)$ 是个体在处理 $A = a$ 下的潜在结局，$A$ 是处理变量，$L$ 是 38 维协变量向量，$e(L)$ 是定义 5.1 给的标量倾向得分；记号 $\perp\!\!\!\perp$ 表示条件独立。等式左边说"在完整 $L$ 的取值层内处理分配像随机的"，等式右边说"在同一倾向得分值 $e(L)$ 的人群里处理分配也像随机的"。这一条把对 38 维向量的条件，压成了对一个标量的条件。

  <img src="../assets/chap05/02_dim_reduction.gif" alt="降维定理：38 维协变量压成一维倾向得分轴" width="720" />
</div>

把这件事翻成临床语言。两位患者倾向得分都是 0.50，意味着不管他们具体的 APACHE、血压、肌酐怎么排列组合，"被插管的概率"汇总下来都是一半一半。在这两位之间比较 RHC 与非 RHC，效果与"在协变量完全相同的两位之间比较"一样——前提是倾向得分模型估对了。原本要在 38 维空间里找邻居的活，现在变成在 0 到 1 数轴上找邻居。

**停下来想一想。** 把 38 个变量压成一个 0 到 1 之间的数字，听起来像把一锅汤过滤成一杯水，信息量肯定有损失。定理给出的承诺范围其实更窄：它只保证"为消除混杂所需的那部分信息"被原样保留下来，结局相关的那部分信息丢了不影响结论。代价从在 38 维空间里精确匹配，转移到把 $e(L)$ 这个函数估对——只要逻辑回归把每位患者的被插管概率算对了，剩下的 38 个变量就不用管了。

## 5.3 用逻辑回归估计倾向得分

降维定理把可行性问题解决了，但真实的 $e(L)$ 是未知的，需要从数据里估。最常用的工具是逻辑回归：把处理变量 $A$ 当因变量，38 个协变量当自变量，拟合 logit 模型，再把每位患者的预测概率拿出来当作估计倾向得分 $\hat e(L_i)$。这个模型与第 3 章、第 4 章里建结局模型用的是同一个 R 包，只是因变量从 `death180_bin` 换成了 `rhc`。

```r
library(tidyverse)
set.seed(2026)

d <- read_csv(here::here("data", "rhc.csv"), show_col_types = FALSE) |>
  mutate(death180_bin = ifelse(death180 == "Yes", 1, 0))

# 38 个协变量,第 2 章 DAG 确定的调整集
covs <- c("age", "sex", "edu", "das_index", "apache_score",
  "glasgow_coma_score", "blood_pressure", "wbc", "heart_rate",
  "respiratory_rate", "temperature", "pa_o2vs_fio2",
  "albumin", "hematocrit", "bilirubin", "creatinine",
  "sodium", "potassium", "pa_co2", "ph", "weight",
  "dnr_status", "medical_insurance", "race", "income",
  "cancer", "cardiovascular", "congestive_hf", "dementia",
  "psychiatric", "pulmonary", "renal", "hepatic",
  "gi_bleed", "tumor", "immunosupperssion", "transfer_hx", "mi")

fml <- as.formula(paste("rhc ~", paste(covs, collapse = " + ")))

# 倾向得分模型,对处理分配机制建模,不是对结局建模
ps_model <- glm(fml, data = d, family = binomial)
d$ps <- predict(ps_model, type = "response")
summary(d$ps)
```

> [!TIP]
> **结果解读**
>
> 倾向得分的均值为 0.381,中位数 0.360,范围从 0.005 到 0.960。RHC 组 2184 人的平均倾向得分为 0.508,对照组 3551 人为 0.303。两组的均值差距 0.205 直接对应着第 2 章看到的"重症患者更容易被插管"的临床事实——RHC 组本来就装着倾向得分偏高的那批人。

估完倾向得分,第一个要做的诊断是把两组的 $\hat e(L)$ 分布画在同一张图上,检查重叠区。重叠区指的是两组分布同时存在的那段倾向得分范围,只有在重叠区里,匹配和加权才有真实邻居可用。

<div align="center">
  <img src="../assets/chap05/03_ps_overlap.gif" alt="RHC 组与对照组的倾向得分分布及重叠区" width="720" />
</div>

对照组的密度峰值落在 0.10 附近,RHC 组的密度峰值落在 0.50 附近。两组有清晰的分离,但重叠区域从 0.05 一直延伸到 0.90 以上,覆盖了绝大多数样本。倾向得分接近 0 的位置几乎只有对照组,接近 1 的位置几乎只有 RHC 组,这两条尾巴是后续正值性讨论的重点。

> [!WARNING]
> **雷区 — 倾向得分模型不追求 AUC**
>
> 不少研究者上手就用 AUC 评价倾向得分模型,AUC 越高就觉得模型越好。这条直觉来自分类任务,放到因果推断里是错的。倾向得分模型的目的不是把"谁会被插管"预测得越准越好,而是匹配或加权之后让协变量在两组之间达到平衡。AUC 接近 1 反而是坏消息——它意味着倾向得分把两组完全分开,几乎没有重叠区,匹配找不到邻居,加权权重直接爆炸。判断倾向得分模型好坏的正确指标是匹配或加权之后每个协变量的标准化均值差是否落到 0.1 以下,本章 §5.6 会展开。

## 5.4 倾向得分匹配 PSM

倾向得分有匹配、加权两类常见用法。匹配是直觉最干净的一种:在对照组里给每位 RHC 患者找一位倾向得分最接近的"双胞胎",组成新的匹配样本,然后直接比较两组结局。

回到那位 50 岁、APACHE 60 的 RHC 患者,他的倾向得分是 0.85。匹配算法会在 3551 名对照组患者里搜索倾向得分最接近 0.85 的那一位——可能是某位 58 岁、APACHE 55、收缩压 80 的没插管患者,他的倾向得分恰好是 0.84。这两位在 38 维原始协变量上不完全一样,但在被插管概率这件事上几乎相同,可以当作"同一类人"。一旦每位 RHC 患者都找到了这样的对照,匹配后样本里两组的倾向得分分布几乎重合,降维定理保证了协变量在两组之间也跟着接近平衡。

> [!NOTE]
> **定义 5.3 — 倾向得分匹配 PSM**
>
> 倾向得分匹配 PSM 把每位处理组个体与倾向得分最接近的一位对照组个体配对成 1:1 匹配样本。常用具体算法是带卡钳的最近邻匹配:
>
> $$ j^*(i) = \arg\min_{j: A_j = 0,\, |\hat e(L_i) - \hat e(L_j)| < c} |\hat e(L_i) - \hat e(L_j)|. $$
>
> 其中 $i$ 是某位 RHC 患者的下标,$j$ 是任意一位对照组患者的下标,$\hat e(L_i)$ 与 $\hat e(L_j)$ 是估出来的倾向得分,$c$ 是卡钳值,Austin 2011 推荐取倾向得分 logit 标准差的 0.2 倍;$j^*(i)$ 是给患者 $i$ 找到的那位对照组双胞胎。卡钳的作用是防止"勉强配对"——某位 RHC 患者倾向得分 0.95,对照组里最近的一位是 0.50,离得太远,就让 RHC 这位被丢弃,不强行凑对。

<div align="center">
  <img src="../assets/chap05/04_ps_matching.gif" alt="PSM 配对动画:每位 RHC 个体在对照组中找最近邻" width="720" />
</div>

匹配可以放回也可以不放回。不放回是默认做法,每位对照组患者最多被使用一次,匹配后样本里没有重复个体;放回则允许同一位对照组患者作为多位 RHC 患者的双胞胎,代价是后续推断需要把权重处理写对。1:1 匹配只给每位 RHC 找一位对照,1:k 匹配给每位 RHC 找 k 位对照,k 越大保留的对照样本越多但匹配质量越差。本章为了与 §5.6 平衡诊断对齐,采用最常见的设定:卡钳 0.2、不放回、1:1 最近邻。

```r
library(MatchIt)

# 1:1 最近邻匹配,卡钳值 0.2 倍 logit PS 标准差
m_out <- matchit(fml, data = d, method = "nearest",
                 distance = "glm", caliper = 0.2, ratio = 1)

# 匹配后样本
m_data <- match.data(m_out)
cat("Matched sample:", nrow(m_data), "\n")
cat("RHC:", sum(m_data$rhc), "  No RHC:", sum(m_data$rhc == 0), "\n")

# 匹配样本上的风险差
rd_psm <- mean(m_data$death180_bin[m_data$rhc == 1]) -
          mean(m_data$death180_bin[m_data$rhc == 0])
cat("PSM Risk Difference:", round(rd_psm, 4), "\n")
```

> [!TIP]
> **结果解读**
>
> 匹配后保留 3670 人,每组 1835 人。原始 2184 名 RHC 患者中有 349 人因卡钳约束未找到配对被丢弃,这些是倾向得分接近 0.95 以上的极重症患者,对照组里没有同样高分的双胞胎。对照组 3551 人中有 1716 人未被任何 RHC 患者选中。匹配样本上的 180 天死亡风险差 RD = 0.076,95% bootstrap CI 为 [0.041, 0.109]——RHC 组比对照组高 7.6 个百分点,方向与第 3 章 OR = 1.34、第 4 章 RD = 0.060 一致。

PSM 把 5735 人砍到 3670 人,丢弃了 36% 样本。被丢弃的两端有不同的临床含义:对照组那 1716 人是低倾向得分的轻症患者,他们没有被 RHC 组选中,因为 RHC 组里没人病情那么轻;RHC 组那 349 人是高倾向得分的极重症患者,对照组里几乎没有同样重的人没被插管。匹配样本估出来的因果效应严格只对应"那 3670 个能找到双胞胎的人",不再代表全部 5735 名 ICU 患者。要把结论推广到全人群,需要追加额外的可迁移性假设。

## 5.5 逆概率加权 IPW

匹配靠丢弃样本来构造可比群体,加权走另一条路:不丢人,而是给每位患者赋一个权重,用权重把样本扭回到一个人造的"伪人群",在这个伪人群里处理分配与协变量独立。具体的权重形式是倾向得分的倒数。

回到那位倾向得分 $\hat e = 0.85$ 的 50 岁 RHC 患者。他实际上接受了处理($A = 1$),逆概率加权给他的权重是 $1 / 0.85 \approx 1.18$,也就是把他算作 1.18 个人。这个 1.18 接近 1,说明"像他这种人本来就高概率被插管,数据里他的同类很多,代表性已经够了,不需要放大"。再设想一位倾向得分 $\hat e = 0.10$ 的轻症患者,他也碰巧接受了 RHC,权重是 $1 / 0.10 = 10$,他一个人在伪人群里被算作 10 个人——因为像他这种轻症在 RHC 组里非常稀有,需要把他的声音放大才能代表"如果全人群都接受 RHC"的情形。对照组的逻辑是对称的,权重用 $1 / (1 - \hat e)$:倾向得分高的对照组个体本来稀有,要被放大;倾向得分低的对照组个体本来普遍,权重接近 1。

> [!NOTE]
> **定义 5.4 — 逆概率加权 IPW**
>
> 估计 ATE 时,IPW 给每位个体的权重为
>
> $$ w_i = \frac{A_i}{\hat e(L_i)} + \frac{1 - A_i}{1 - \hat e(L_i)}. $$
>
> 其中 $A_i$ 是患者 $i$ 的处理状态,$\hat e(L_i)$ 是估出来的倾向得分;$A_i = 1$ 时第一项激活、第二项为零,权重等于 $1/\hat e(L_i)$;$A_i = 0$ 时反过来,权重等于 $1/(1-\hat e(L_i))$。把每位个体按 $w_i$ 加权汇总之后得到的伪人群,在协变量分布上重现全人群,处理分配在伪人群里与 $L$ 独立,等价于一个伪随机试验。

<div align="center">
  <img src="../assets/chap05/05_ipw_weights.gif" alt="IPW 权重柱图与极端权重雷区" width="720" />
</div>

```r
library(WeightIt)

# IPW:估计 ATE
w_ipw <- weightit(fml, data = d, method = "glm", estimand = "ATE")
summary(w_ipw)

d$w_ipw <- w_ipw$weights

# 加权后的风险差
ate_ipw <- weighted.mean(d$death180_bin[d$rhc == 1], d$w_ipw[d$rhc == 1]) -
           weighted.mean(d$death180_bin[d$rhc == 0], d$w_ipw[d$rhc == 0])
cat("IPW Risk Difference:", round(ate_ipw, 4), "\n")
```

> [!TIP]
> **结果解读**
>
> IPW 估计的 ATE 风险差为 RD = 0.055,95% bootstrap CI 为 [0.025, 0.085]。全部 5735 个样本都保留在分析里。权重分布显示:处理组权重范围 1.04 到 26.45,对照组 1.01 到 14.79。有效样本量为处理组 1319、对照组 2806,相比原始的 2184 与 3551 都有明显下降——这就是极端权重稀释信息的代价。

IPW 给的 RD = 0.055 比 PSM 的 0.076 略低,但方向一致,RHC 仍然提高 180 天死亡率。两个数字不应该完全一样:PSM 估的是匹配后那 3670 人的效应,IPW 估的是全部 5735 人的边际 ATE,目标人群本身不同。

权重最大值 26.45 是 IPW 的硬伤。具体到那位 RHC 患者,他的倾向得分大约是 1/26.45 ≈ 0.038,意味着根据他的基线特征,被插管的概率不到 4%,但他偏偏被插了。这种极少数"反预期接受处理"的个体在伪人群里被放大了 26 倍,他一个人主导着 IPW 估计的局部行为。再极端一点,如果某位患者倾向得分 $\hat e = 0.98$、属于"几乎必然被插管"的极重症人群,他作为 RHC 接受处理时权重是 $1 / 0.98 \approx 1.02$,没问题;但只要这位极重症患者碰巧没被插管,权重就变成 $1 / (1 - 0.98) = 50$,一个人被算作 50 个人。倾向得分越靠近 0 或 1 的尾部,IPW 越脆弱。

> [!WARNING]
> **雷区 — 极端权重让 IPW 方差爆炸**
>
> IPW 的方差对极端权重高度敏感。一个倾向得分 0.98 的患者一旦落入与其预测相反的组,权重直接变成 50,他一个人贡献的信息量等于 50 个普通患者。RHC 数据里这种极端值不多但确实存在,有效样本量从 5735 降到处理组 1319、对照组 2806,差不多是被这少数极端权重稀释掉的。诊断方法是检查权重最大值与有效样本量比例:最大权重超过 20 就要警惕,有效样本量低于原始样本量一半就说明极端权重在主导估计。常见的应对策略有两种:一是权重截断,把超过某个阈值的权重硬性截到阈值水平,代价是引入一点偏差换取方差大幅下降;二是改用稳定权重 SW,用边际处理概率 $P(A=a)$ 替换分子里的 1,把权重均值锁回到 1。两种做法都不能从根本上解决正值性接近违反的问题,只能让症状缓和。

## 5.6 平衡诊断:SMD 与 Love plot

倾向得分模型估出来了,匹配做完或权重赋好了,效应估计也算出来了——但还有一件事没做:验证两组在协变量上是不是真的平衡。匹配或加权之后协变量分布该接近相等,如果不接近,就说明倾向得分模型估错了,前面所有的 RD 都不可信。这一步叫平衡诊断。

诊断指标用标准化均值差 SMD,而不是 t 检验或 $p$ 值。原因是 $p$ 值同时依赖效应大小与样本量,RHC 数据 5735 人量级大,即使两组某个协变量均值只差 0.05 个标准差,$p$ 值也能压到 0.001 以下,看上去不平衡其实可以忽略。SMD 把样本量摘掉,直接报"差了几个标准差"。

> [!NOTE]
> **定义 5.5 — 标准化均值差 SMD**
>
> $$ \text{SMD} = \frac{\bar X_1 - \bar X_0}{\sqrt{(s_1^2 + s_0^2) / 2}}. $$
>
> 其中 $\bar X_1$ 与 $\bar X_0$ 分别是处理组和对照组在某个协变量上的均值,$s_1^2$ 与 $s_0^2$ 是对应的样本方差。分母是把两组方差取均值再开根,得到一个综合标准差刻度。SMD 的绝对值小于 0.1 通常被视为良好平衡的标志(Austin 2011);介于 0.1 与 0.2 之间属于轻度不平衡,可以接受但需要在论文里报;超过 0.2 视为不平衡,提示倾向得分模型需要调整。

Love plot 是把所有协变量的 SMD 同时画在一张图上的可视化工具,纵轴列协变量名字,横轴是 SMD 绝对值,在 0.1 处画一条参考竖线。每种调整方法用一种颜色和形状,所有点都落到 0.1 竖线左边就说明该方法平衡得好。

```r
library(cobalt)

# Love plot:同时展示未调整 / PSM / IPW 三种状态的协变量平衡
love.plot(fml, data = d,
          stats = "m", abs = TRUE,
          thresholds = c(m = 0.1),
          weights = list(PSM = m_out, IPW = w_ipw),
          colors = c("#999999", "#EF6548", "#4292C6"),
          shapes = c(17, 16, 15),
          sample.names = c("Unadjusted", "PSM", "IPW"))
```

> [!TIP]
> **结果解读**
>
> 未调整时,`apache_score` 的 SMD 达到 0.48,`blood_pressure` 达到 0.49,两个变量本身就承担了 RHC 数据里最重的混杂作用——重症与低血压是医生决定插管的两个主信号。`pa_o2vs_fio2`、`hematocrit`、`pa_co2`、`weight`、`creatinine` 几个变量 SMD 也都超过 0.20。PSM 之后绝大多数变量 SMD 降到 0.05 以下,但 `apache_score` 仍有 0.08 的残余不平衡,逼近 0.1 阈值;说明 1:1 最近邻在极重症层面找邻居仍有难度。IPW 几乎所有变量 SMD 都低于 0.05,平衡效果优于 PSM。

> [!WARNING]
> **雷区 — 平衡诊断看协变量分布,不看 Hosmer-Lemeshow**
>
> 论文里常见的错误是:报告倾向得分模型 Hosmer-Lemeshow 拟合优度检验通过、$p > 0.05$,就认为倾向得分方法可靠,不再检查匹配或加权之后的协变量平衡。模型拟合好不代表平衡好——倾向得分模型即使整体拟合 $p$ 值漂亮,某个关键协变量在 PSM 之后可能仍然有 SMD = 0.15 的残余不平衡。诊断的正确流程是:拟合倾向得分模型,实施 PSM 或 IPW,然后逐变量查 SMD。如果某些变量 SMD 仍超过 0.1,说明倾向得分模型设定有问题,常见的修补办法是加入这些变量与年龄、APACHE 的交互项,或对连续变量加二次项与样条。平衡诊断本身是迭代的,直到所有协变量 SMD 低于阈值才算完。

正值性是这一节最后要讨论的假设。倾向得分方法的三大前提是可交换性、正值性、一致性。正值性要求每一种协变量组合下接受处理和不接受处理的概率都严格大于零,即 $0 < e(L) < 1$。RHC 数据里倾向得分最小值 0.005、最大值 0.960,没有真正的 0 或 1 出现,正值性技术上没被硬违反。但 0.005 已经非常接近零——意味着有些患者根据基线特征几乎不可能接受 RHC,他们的因果效应本质上无法从观察数据可靠估计。这种"近乎违反正值性"的情况在 IPW 里直接体现为权重接近 200 的极端值,在 PSM 里体现为找不到卡钳内的对照。诊断方法很简单:画倾向得分分布图,看两端尾巴是否有明显单组样本;在论文讨论里诚实说明结论不适用于这些极端人群。

四种方法的方向完全一致:RHC 提高 180 天死亡率。回归给的 OR = 1.34 与三个 RD 不在一个尺度上,但读成"死亡几率比升高 34%"与"死亡风险差升高 5.5 到 7.6 个百分点"对应得上。G 计算的 RD = 0.060 与 PSM 的 0.076、IPW 的 0.055 同处一个数量级,数值差异主要来源有两条:一是 PSM 与 IPW 的目标人群不同,PSM 估的是匹配后的 3670 人,IPW 估的是全部 5735 人;二是建模对象不同,G 计算建结局模型,PSM 与 IPW 建处理模型,两个模型即便理论上识别同一个因果量,有限样本下的偏差和方差也不会完全相等。

到这里,前几章的方法可以分成两派:G 计算赌结局模型对,倾向得分方法赌处理模型对,每一派都只有一条保险绳。下一章引入 AIPW 双重稳健估计——它同时建结局模型与处理模型,只要其中一个对就给出一致估计,两条绳只要一根不断就安全。AIPW 还有一个深层结果:即便两个模型都用机器学习这种收敛速度比参数模型慢的工具拟合,只要满足"乘积速率"条件,AIPW 估计仍然能达到根号 n 收敛,这是 TMLE 与 DML 等一系列现代方法的统一基石,也是第 7 章把机器学习接入因果推断的入场券。

## 本章知识地图

<div align="center">

| 核心概念 | 核心内容 | 常见误解 | 为什么错 |
|:--|:--|:--|:--|
| **倾向得分** | 给定 $L$ 的接受处理条件概率,把 38 维协变量压成一个 0 到 1 的标量 | 倾向得分是处理效应 | 倾向得分只描述处理分配机制,与结局 $Y$ 无关,要再走匹配或加权这一步才能拿到效应 |
| **降维定理** | 按 $e(L)$ 调整等价于按全部 $L$ 调整 | 倾向得分包含了结局信息 | 定理只保证条件独立性传递到 $e(L)$,不建模结局 |
| **PSM** | 按倾向得分最近邻 1:1 配对,组成可比样本后比较结局 | 匹配后不需要再检查平衡 | 卡钳不能保证所有协变量 SMD 都达标,必须用 SMD 逐变量诊断 |
| **IPW** | 按倾向得分倒数加权,构造协变量平衡的伪人群 | AUC 越高的倾向得分模型越好 | 模型好坏看加权后协变量平衡,AUC 接近 1 反而意味着重叠区被压扁 |
| **极端权重** | 倾向得分接近 0 或 1 时,$1/\hat e$ 或 $1/(1-\hat e)$ 飙到 50 以上 | 加权后样本量没变就没问题 | 有效样本量比原始样本量低一半就说明极端权重在主导 |
| **SMD 与 Love plot** | 标准化均值差衡量两组协变量差几个标准差,Love plot 把所有变量画在 0.1 阈值线两侧 | 用 $p$ 值或 t 检验判断平衡 | $p$ 值随样本量放大,RHC 这种 5735 量级里小不平衡也会显著 |
| **正值性接近违反** | $\hat e(L)$ 进入 0.005 或 0.96 的尾部,理论上没违反但方差实际爆炸 | 没出现 $\hat e = 0$ 或 1 就安全 | 接近 0 或 1 已经足以让 IPW 权重达到 50 以上,PSM 找不到卡钳内邻居 |
